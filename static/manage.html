Hỗ trợ proxy HTTP và SOCKS5

Mục tiêu kiểm tra: https://labs.google/

Cấu hình sinh nội dung

Thời gian chờ sinh ảnh, phạm vi 60-3600 giây (1 phút - 1 giờ); quá hạn sẽ tự giải phóng khóa Token

Thời gian chờ sinh video, phạm vi 60-7200 giây (1 phút - 2 giờ); quá hạn sẽ trả về lỗi timeout từ API upstream

Số lần thử lại tối đa khi request sinh/tải lên/polling thất bại, tối thiểu là 1

Cấu hình luân chuyển Token

Luân chuyển ngẫu nhiên dùng chiến lược ưu tiên theo tải mặc định; luân chuyển tuần tự sẽ dùng lần lượt theo thứ tự ổn định của các Token khả dụng, hết một vòng mới bắt đầu vòng mới.

Cấu hình xử lý lỗi

Token sẽ tự bị vô hiệu hóa sau khi lỗi liên tiếp đạt số lần này

Cấu hình Cache

Khi tắt, ảnh và video được tạo sẽ trả thẳng URL gốc và không cache về local

Thời gian hết hạn của file cache, phạm vi 0-86400 giây; 0 nghĩa là không bao giờ hết hạn, không tự xóa file cache

Để trống sẽ dùng địa chỉ máy chủ, ví dụ: https://yourdomain.com

Cấu hình kết nối plugin

Tiện ích mở rộng Chrome cần cấu hình endpoint này

Dùng để xác thực plugin Chrome; để trống sẽ tự tạo token ngẫu nhiên

Khi plugin cập nhật token, nếu token đang bị vô hiệu sẽ tự bật lại

ℹ️ Hướng dẫn: Sau khi cài plugin Chrome, nhập endpoint kết nối và Token vào plugin; plugin sẽ tự động trích xuất cookie Google Labs và đẩy lên hệ thống

Dùng để tự động lấy mã reCAPTCHA

Dùng để tự động lấy mã reCAPTCHA

Địa chỉ dịch vụ EzCaptcha, mặc định: https://api.ez-captcha.com

ℹ️ Giải Captcha qua trình duyệt: Dùng Playwright điều khiển trình duyệt để lấy mã, không cần cấu hình thêm nhưng tốn tài nguyên hệ thống hơn

Cấu hình proxy riêng cho trình duyệt có hiển thị

✅ Hỗ trợ: proxy HTTP/HTTPS/SOCKS5/SOCKS5H, đều hỗ trợ xác thực
Ví dụ: http://user:pass@proxy.com:8080 hoặc socks5://user:pass@proxy.com:1080

Số instance trình duyệt chạy đồng thời; mỗi trình duyệt chỉ mở 1 tab, yêu cầu được phân phối luân phiên

ℹ️ Giải Captcha bằng trình duyệt tích hợp: Dùng nodriver điều khiển trình duyệt, hỗ trợ tái sử dụng tab, hiệu năng tốt hơn

Chỉ ảnh hưởng số lượng project_id mà Token này có thể luân chuyển, không quyết định số tab giải Captcha.
Tab giải Captcha dùng chung qua pool toàn cục.

Quyết định số tab giải Captcha dùng chung mà chế độ personal giữ lại.
Khả năng giải đồng thời phụ thuộc vào giá trị này; thiếu thì ưu tiên tăng lên.

Sau bao nhiêu giây tab rảnh sẽ tự thu hồi, mặc định 600 giây (10 phút)

Cấu hình proxy riêng cho trình duyệt tích hợp (ưu tiên cao hơn proxy yêu cầu toàn cục)

✅ Hỗ trợ: proxy HTTP/HTTPS/SOCKS5/SOCKS5H, đều hỗ trợ xác thực
Ví dụ: http://user:pass@proxy.com:8080 hoặc socks5://user:pass@proxy.com:1080

ℹ️ Giải Captcha từ xa: Gọi dịch vụ giải Captcha độc lập qua HTTP để lấy token, sau khi xong sẽ callback finish/error

Ví dụ: http://127.0.0.1:8060
